# Personal Skill

## Purpose
Help me solve programming tasks with minimal, precise, and test-validated changes.

## Rules
1. Prefer small, focused edits.
2. Preserve existing behavior unless change is requested.
3. Run relevant checks/tests for touched areas.
4. Explain assumptions and limitations clearly.
5. Avoid introducing secrets or insecure patterns.

## Output style
- Be concise.
- Show changed files.
- Include verification steps and results.
